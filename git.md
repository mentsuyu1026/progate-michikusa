# よくある操作
## 新しい機能を実装開始する場合
1. mainブランチに移動する
   - `git checkout main`
2. ローカルリポジトリのmainブランチを更新する
   - `git pull origin main`
3. mainブランチから分岐して、その機能のためのブランチを作成する
   - `git checkout -b <ブランチ名>`
   - 本プロジェクトにおいて、ブランチ名は`feature/<作業者名>/<機能名>`とする
   - コマンド例: `git checkout -b feature/tombo/score`
4. コードなどの編集を開始する

## 作成した変更をリモートリポジトリに反映する場合
1. (別ディレクトリにいる場合)プロジェクトルートに移動する
   - `cd <プロジェクトルート>`
     - 本プロジェクトでは、air-hockeyというディレクトリになる。
     - プロジェクトルートでは、`ls -force`を実行したとき、`.git`というディレクトリが確認できる。
2. リモートリポジトリから、最新のmainブランチを取り込む
   - `git pull origin main`
3. 変更をステージングする
   - `git add .`
4. ステージングした変更から、コミットを作成する
   - `git commit -m "<コミット名>"`
   - 本プロジェクトにおいて、コミット名は`<プレフィックス>: <内容>`とする
     - プレフィックスは、コミットの分類。機能追加なら`add`、バグ修正なら`fix`など。
   - コマンド例: `git commit -m "add: スコア表示"`
5. コミットを、リモートリポジトリの現在のブランチに反映させる
   - `git push origin <ブランチ名>`
     - ブランチ名は以下のコマンドで確認できる
       - `git branch`
6. GitHubで、[プロジェクトのリモートリポジトリ](https://github.com/tombo-akizu/air-hockey)を開く。
7. GitHubのWebページを操作する。
   - Create PullRequestボタンからプルリクエストを作成し、マージする。


# Terms
- 変更
  - ファイルへの編集を、行単位で表現したデータ。
  - 例:

```cs
int x = 2;
int y = x + 2;
int z = y;
```

を

```cs
int x = 2;
int y = x * 2;
Debug.Log(y);
```

に書き換えた場合、変更は

```diff
int x = 2;
- int y = x + 2;
+ int y = x * 2;
- int z = y;
+ Debug.Log(y);
```

と表現される。追加された行が`+`、削除された行が`-`で表現され、行の内容が書き換わった場合、削除の後で追加したものとして表現される。

- ステージング
  - 変更をgitシステムに渡すこと。
- コミット
  - 変更をまとめ、名前を付けたもの。
  - 「機能Xの実装」「バグYの修正」など、変更の目的1つに対してコミットを1つ作ることが望ましい。

- リポジトリ
  - gitが管理するファイル・ディレクトリ群。
- リモートリポジトリ
  - [GitHub](https://github.com)上にアップロードされたリポジトリ。
- ローカルリポジトリ
  - 各コンピュータに保存されたリポジトリ。
  - プロジェクトメンバーは、自身のローカルリポジトリ上でコミットを作成し、リモートリポジトリに反映させることで、プロジェクトを進行する。
- ブランチ
  - プロジェクトは、コミット(変更の塊)を積み上げることで進行していく。
  - このコミットの連なりを分岐させることができる。
  - 分岐したそれぞれの連なりを、ブランチという。
- mainブランチ
  - プロジェクトの本流となるブランチ。
- マージ
  - 現在のブランチに作成したコミットを、別のブランチに取り込むこと。
  - 本プロジェクトでは基本的にmainブランチに取り込む。
- プルリクエスト
  - マージするためのリクエスト。
  - エンジニアの現場では、作業者はプルリクエストの作成までを行う。
  - プロジェクトマネージャなどが、プルリクエストを確認し、問題なければマージする。

# Commands
## branch関連
- `git branch`
  - ブランチの一覧を表示する。
- `git checkout <ブランチ名>`
  - 指定したブランチに移動する。
- `git checkout -b <ブランチ名>`
  - 現在のブランチから分岐して、指定した名称のブランチを作成する。

## 書き込み関連
- `git add <ファイル/ディレクトリ>`
  - ファイル/ディレクトリへの変更をステージングする。
- `git commit -m "<メッセージ>"`
  - ステージングした変更をまとめて、コミットを作る。
- `git push origin <ブランチ名>`
  - 作成したコミットを、リモートリポジトリの指定したブランチに反映させる。

## 読み込み関連
- `git clone <リモートリポジトリURL>`
  - リモートリポジトリをダウンロードし、ローカルリポジトリを作成する。
  - 最初に一度だけ行う。
- `git pull origin <ブランチ名>`
  - リモートリポジトリの指定したブランチのコミットを、ローカルブランチに取り込む。
